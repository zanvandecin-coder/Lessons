Разберём проблему и решение пошагово — похоже, возникли сложности с подключением к GitHub и установкой `nvm`.

## Почему возникла ошибка

1. **Ошибка подключения к GitHub** (`Could not resolve host: githubusercontent.com`):
   - проблемы с интернет‑соединением;
   - неполадки с DNS (не удаётся преобразовать доменное имя в IP‑адрес);
   - временные проблемы на стороне GitHub.
2. **Команда `nvm` не найдена**:
   - `nvm` ещё не установлен;
   - после установки не выполнен перезапуск терминала или обновление окружения (`source ~/.bashrc`).

------

## Пошаговое решение

### Шаг 1. Проверьте интернет‑соединение и DNS

Выполните команды:

bash

```bash
ping -c 3 google.com
ping -c 3 github.com
```

- Если обе команды работают — проблема, скорее всего, временная. Попробуйте повторить установку позже.
- Если `google.com` пингуется, а `github.com` — нет, проблема с DNS. Попробуйте сменить DNS‑серверы на публичные (например, Google DNS: `8.8.8.8` и `8.8.4.4`).
- Если ни один хост не пингуется — проверьте подключение к интернету (кабель, Wi‑Fi, настройки сети).

### Шаг 2. Установите `nvm` альтернативным способом

Если проблема с доступом к `githubusercontent.com`, используйте официальный скрипт установки с другого источника:

1. Скачайте скрипт установки:

bash

```bash
wget -qO- https://raw.githubusercontent.com/creationix/nvm/v0.39.7/install.sh | bash
```

(Замените `v0.39.7` на актуальную версию `nvm`, если нужно.)

1. Обновите окружение, чтобы команда `nvm` стала доступна:

bash

```bash
source ~/.bashrc
```

Или просто перезапустите терминал.

1. Проверьте, что `nvm` установлен:

bash

```bash
nvm --version
```

Если команда выводит номер версии (например, `0.39.7`), установка прошла успешно.

### Шаг 3. Установите и активируйте Node.js версии 22

Теперь, когда `nvm` доступен, выполните:

bash

```bash
nvm install 22.12.0
nvm use 22.12.0
```

Это:

- скачает и установит Node.js версии 22,12,0;
- сделает эту версию активной в текущей сессии терминала.

### Шаг 4. Проверьте версии Node.js и npm

Убедитесь, что всё работает:

bash

```bash
node --version
npm --version
```

Ожидаемый результат:

- `node --version` → `v22.12.0` или выше;
- `npm --version` → актуальная версия npm для Node.js 22.

### Шаг 5. Настройте `nvm` для автоматического использования версии 22 при запуске терминала

Чтобы не вводить `nvm use` каждый раз, задайте версию по умолчанию:

bash

```bash
nvm alias default 22.12.0
```

Теперь при открытии нового терминала будет автоматически использоваться Node.js 22,12,0.
